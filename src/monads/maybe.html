Monad: Maybe[a]

Commonly represents a value that may, or may not be available. (In some languages, the Maybe monad is called Option or Optional).

Maybe[a] allows one to model, explicitly, values that may not be always present (i.e.: nullable values), without the problems associated with using null and undefined. Furthermore, Maybe[a] values can be composed in manners that are similar to other monads, by using the generic sequencing and composition.

There is two possibilities in the Maybe[a] monad. Either we have a value, indicated by the constructor Just(a), or we don't have a value, indicated by the constructor Nothing(). As such, the Maybe[a] monad may be used safely to encode any case where it's possible to either have a value or not, without the possibility of ambiguity.

Another interesting property of this monad is that, since you need to explicitly wrap and unwrap your values, you can't have a NullPointerException or TypeError by using things that don't exist (e.g.: a library broke your expectations of the code behaviour, or you forgot to check for null values). This way, using a Maybe[a] monad reduces the number of runtime errors.
